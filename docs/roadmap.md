# Roadmap

## MVP

MVP закрывает полный соревновательный N-back цикл.

| Задача | Результат |
| --- | --- |
| Авторизация | Пользователи могут регистрироваться, входить и иметь активные сессии. |
| Создание комнаты | Игрок может создать multiplayer-комнату с настройками N-back. |
| Присоединение к комнате | 2-4 игрока могут зайти до старта игры. |
| Серверная последовательность | Все игроки получают одни и те же авторитетные стимулы. |
| Серверная проверка ответа | Корректность вычисляется только на сервере. |
| Scoreboard | Игроки видят верные ответы, ошибки и штрафы. |
| Штрафное ускорение | Каждая третья ошибка ускоряет игру для всех. |
| Расчёт победителя | Завершённые игры дают детерминированную таблицу результатов. |
| История игр | Завершённые игры сохраняются и доступны пользователям. |

## Next Features

| Задача | Зачем |
| --- | --- |
| Улучшение reconnect recovery | Повышает надёжность при нестабильной сети. |
| Режим наблюдателя | Позволяет смотреть игру без влияния на счёт. |
| Invite links | Ускоряет подключение игроков к комнате. |
| Фильтры истории турниров | Упрощают анализ соревновательных результатов. |
| Подробная история ответов | Даёт post-game анализ по каждому стимулу. |
| Rate limiting | Защищает production-инфраструктуру от спама. |

## Long-Term Vision

| Задача | Зачем |
| --- | --- |
| Ranked matchmaking | Создаёт более честные соревновательные сессии. |
| Сезонные лидерборды | Добавляют долгосрочную мотивацию. |
| Расширенные игровые режимы | Увеличивают вариативность без отказа от server-authoritative модели. |
| Accessibility improvements | Делают тренажёр доступнее для большего числа игроков. |
| Analytics dashboard | Помогает оценивать прогресс игроков и баланс игры. |
