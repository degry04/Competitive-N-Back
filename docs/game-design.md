# Игровой дизайн

## Общая модель

Competitive N-Back поддерживает несколько когнитивных тренажеров в общей multiplayer-комнате. Все игроки получают одинаковые стимулы, а сервер:

- определяет текущий стимул;
- принимает или игнорирует ответы;
- считает очки и ошибки;
- обновляет метрики;
- ускоряет игру после штрафов.

## Поддерживаемые режимы

| Режим | Основное правило | Что считается победой |
| --- | --- | --- |
| N-back | Нажать, если текущая клетка совпадает с клеткой N ходов назад | Больше очков за верные ответы |
| Recent-5 | Нажать, если клетка уже была среди последних 5 стимулов | Больше очков |
| Go / No-Go | Нажимать только на GO, NO_GO игнорировать | Больше очков и accuracy |
| Reaction Time | Ждать сигнала и кликнуть быстрее всех | Очко получает самый быстрый корректный игрок |
| Stroop Test | Отвечать по цвету текста, а не по слову | Очко получает самый быстрый правильный ответ |

## N-back

Игроки видят последовательность клеток в сетке 3x3:

```text
0 1 2
3 4 5
6 7 8
```

Для классического режима:

```text
sequence[currentIndex] === sequence[currentIndex - n]
```

Если `currentIndex < n`, совпадение невозможно.

### Пример 2-back

```text
index:    0  1  2  3  4  5
position: 4  1  4  7  4  7
```

| Индекс | Совпадение | Действие |
| --- | --- | --- |
| 0 | нет | не нажимать |
| 1 | нет | не нажимать |
| 2 | да | нажать |
| 3 | нет | не нажимать |
| 4 | да | нажать |
| 5 | да | нажать |

## Recent-5

Игрок нажимает, если текущая клетка встречалась среди последних пяти стимулов.

Пример:

```text
0 1 2 3 4 0
```

Последний `0` считается совпадением, потому что он уже был среди пяти предыдущих стимулов.

## Go / No-Go

Сервер генерирует стимулы двух типов:

- `GO`
- `NO_GO`

Соотношение настраивается при создании комнаты, например `70% GO / 30% NO_GO`.

### Валидация

| Ситуация | Результат |
| --- | --- |
| GO и игрок нажал | correct |
| GO и игрок не нажал | miss |
| NO_GO и игрок не нажал | correct |
| NO_GO и игрок нажал | false positive |

### Метрики

- reaction time;
- false positives;
- misses;
- accuracy.

## Reaction Time

На каждом стимуле сервер сначала ждёт случайную задержку от `1000` до `3000` мс, после чего показывает сигнал.

### Валидация

```text
reactionTime = clickTime - stimulusTime
```

| Ситуация | Результат |
| --- | --- |
| клик до сигнала | false start + penalty |
| клик после сигнала | valid reaction |
| несколько кликов | учитывается только первый |
| клик после timeout | игнорируется |

### Метрики

- average reaction time;
- best time;
- false starts;
- consistency.

### Победа на стимуле

Очко получает самый быстрый корректный игрок.

## Stroop Test

Стимул имеет форму:

```ts
type StroopStimulus = {
  word: string;
  color: string;
};
```

Игрок должен отвечать по цвету текста, а не по слову.

### Пример

```json
{
  "word": "red",
  "color": "blue"
}
```

Правильный ответ: `blue`.

### Типы стимулов

- congruent: `word === color`
- incongruent: `word !== color`

### Валидация

| Ситуация | Результат |
| --- | --- |
| `answer === stimulus.color` | correct |
| другой ответ | mistake |

### Метрики

- reaction time;
- accuracy;
- conflict error rate.

### Победа на стимуле

Очко получает самый быстрый правильный игрок.

## Система ошибок и скорости

Ошибка игрока:

1. увеличивает `errors`;
2. увеличивает `penalty`;
3. каждая третья ошибка уменьшает `currentIntervalMs` для всех игроков.

Формула:

```text
currentIntervalMs = max(minIntervalMs, round(currentIntervalMs * 0.9))
```

## Общие edge cases

| Ситуация | Поведение |
| --- | --- |
| duplicate click | игнорируется |
| late response | игнорируется |
| false start в Reaction Time | штрафуется |
| ответ от игрока вне комнаты | отклоняется |
| старт комнаты с < 2 игроков | отклоняется |
| ручное завершение владельцем | раунд завершается немедленно |

## Rated matches

- Lobby owner can mark a match as `rated`.
- ELO is recalculated only for rated matches.
- Rating changes are written to `rating_history`.
- Bots are not allowed in rated matches.
- Rating and leaderboard are shown on a dedicated stats page instead of match history.
