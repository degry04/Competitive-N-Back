# Игровой дизайн

## Основное правило

Игроки видят последовательность позиций в сетке 3x3. Каждая позиция представлена числом от `0` до `8`.

```text
0 1 2
3 4 5
6 7 8
```

В классическом режиме N-back игрок должен ответить, когда:

```text
sequence[currentIndex] === sequence[currentIndex - n]
```

Если `currentIndex < n`, классическое N-back совпадение невозможно.

## Структура стимула

| Поле | Описание |
| --- | --- |
| `index` | Позиция стимула в последовательности. |
| `position` | Клетка сетки от `0` до `8`. |
| `shownAt` | Серверное время появления стимула. |
| `intervalMs` | Текущий интервал показа. |

Пример:

```json
{
  "index": 7,
  "position": 4,
  "shownAt": "2026-04-23T10:00:08.400Z",
  "intervalMs": 1200
}
```

## Пример: 2-back

Последовательность:

```text
index:    0  1  2  3  4  5
position: 4  1  4  7  4  7
```

Для `n = 2`:

| Индекс | Текущий | 2-back | Совпадение | Ожидаемое действие |
| --- | ---: | ---: | --- | --- |
| 0 | 4 | нет | нет | Не отвечать |
| 1 | 1 | нет | нет | Не отвечать |
| 2 | 4 | 4 | да | Отправить ответ |
| 3 | 7 | 1 | нет | Не отвечать |
| 4 | 4 | 4 | да | Отправить ответ |
| 5 | 7 | 7 | да | Отправить ответ |

## Верные и неверные ответы

### Верный ответ

Состояние:

```text
n = 3
index = 6
sequence[6] = 2
sequence[3] = 2
```

Игрок отправляет ответ. Сервер считает его верным.

Результат:

```json
{
  "isCorrect": true,
  "correct": 5,
  "errors": 1,
  "penalty": 1
}
```

### Неверный ответ

Состояние:

```text
n = 2
index = 5
sequence[5] = 8
sequence[3] = 1
```

Игрок отправляет ответ. Сервер считает его ошибкой.

Результат:

```json
{
  "isCorrect": false,
  "correct": 4,
  "errors": 2,
  "penalty": 2
}
```

## Система ошибок

Ошибка возникает, когда игрок отправляет ответ, но текущий стимул не является совпадением.

При ошибке:

1. `errors` игрока увеличивается на `1`.
2. `penalty` игрока увеличивается на `1`.
3. Если общее число ошибок игрока делится на `3`, игра ускоряется для всех.

## Влияние ошибок на скорость

Каждая третья ошибка конкретного игрока уменьшает общий интервал игры.

| Ошибки игрока | Изменение интервала |
| ---: | --- |
| 1 | Скорость не меняется |
| 2 | Скорость не меняется |
| 3 | Игра ускоряется |
| 4 | Скорость не меняется |
| 5 | Скорость не меняется |
| 6 | Игра ускоряется |

Рекомендуемая формула:

```text
currentIntervalMs = max(minIntervalMs, round(currentIntervalMs * 0.9))
```

Ускорение применяется глобально, потому что игра соревновательная: ошибки одного игрока повышают давление на всех.

## Условия победы

Игра заканчивается, когда:

- достигнут последний стимул;
- владелец комнаты завершил игру вручную.

Определение победителя:

1. Больше `correct` — выше место.
2. При равенстве меньше `errors` — выше место.
3. При полном равенстве используется стабильный порядок присоединения.

Пример:

| Место | Игрок | Верно | Ошибки |
| ---: | --- | ---: | ---: |
| 1 | Alice | 12 | 2 |
| 2 | Bob | 12 | 4 |
| 3 | Carol | 9 | 1 |

Alice побеждает: у неё столько же верных ответов, сколько у Bob, но меньше ошибок.

## Edge Cases

| Ситуация | Ожидаемое поведение |
| --- | --- |
| Ответ до старта | Отклонить с `GAME_NOT_RUNNING`. |
| Ответ после завершения | Отклонить с `GAME_NOT_RUNNING`. |
| Ответ от пользователя вне комнаты | Отклонить с `PLAYER_NOT_IN_ROOM`. |
| Повторный ответ на тот же стимул | Отклонить с `DUPLICATE_ANSWER`. |
| В комнате меньше 2 игроков | Отклонить старт с `ROOM_NOT_READY`. |
| Комната уже запущена | Запретить изменения, допустимые только в lobby. |
| Часы клиента отличаются от серверных | Серверное время имеет приоритет. |
| Переподключение WebSocket | Клиент получает свежий snapshot и продолжает отображение. |
