# Competitive N-Back

Competitive N-Back — веб-приложение для соревновательной тренировки рабочей памяти. Игроки заходят в одну комнату, видят общую последовательность стимулов в сетке 3x3 и нажимают кнопку ответа, когда текущий стимул совпадает со стимулом N шагов назад.

Сервер управляет игровым состоянием: генерирует последовательность, проверяет ответы, применяет штрафы, ускоряет игру и определяет победителя. Клиент только отображает состояние и отправляет намерение игрока.

## Ключевые возможности

| Возможность | Описание |
| --- | --- |
| Комнаты на 2-4 игрока | Несколько игроков участвуют в одной синхронизированной игровой сессии. |
| Серверная последовательность | Стимулы генерируются на сервере и одинаковы для всех игроков. |
| Server-authoritative модель | Сервер проверяет каждый ответ и хранит единственный источник истины. |
| Система ошибок | Ошибки считаются отдельно для каждого игрока. Каждая третья ошибка ускоряет игру для всех. |
| Общий темп игры | Все игроки видят один и тот же индекс стимула и один интервал. |
| Подсчёт победителя | Побеждает игрок с наибольшим числом верных ответов; при равенстве учитывается меньшее число ошибок. |
| Авторизация | Аккаунты и сессии обслуживаются через better-auth. |
| Realtime-обновления | Состояние игры доставляется через tRPC subscriptions поверх WebSocket. |

## Быстрый старт

### Требования

- Node.js 20+
- npm
- PostgreSQL 15+

### Установка

```bash
npm install
```

### Переменные окружения

Создайте `.env.local`:

```bash
DATABASE_URL="postgresql://user:password@localhost:5432/competitive_nback"
BETTER_AUTH_SECRET="replace-with-a-long-random-secret"
BETTER_AUTH_URL="http://localhost:3000"
NEXT_PUBLIC_APP_URL="http://localhost:3000"
```

### Миграции

```bash
npm run db:migrate
```

### Запуск в разработке

```bash
npm run dev
```

Откройте:

```text
http://localhost:3000
```

### Production-сборка

```bash
npm run build
npm start
```

## Обзор потока

1. Авторизованный пользователь создаёт комнату с настройками N-back.
2. Другие игроки присоединяются к комнате.
3. Владелец комнаты запускает игру.
4. Сервер генерирует и продвигает последовательность стимулов.
5. Клиенты подписываются на обновления и отображают текущую клетку.
6. Игроки отправляют ответы.
7. Сервер проверяет ответы, обновляет счёт, применяет штрафы и рассылает изменения.
8. Игра заканчивается после последнего стимула или ручного завершения владельцем.
9. Сервер рассчитывает и сохраняет результаты.
