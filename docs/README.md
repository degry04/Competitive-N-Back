# Competitive N-Back

Competitive N-Back — веб-приложение для соревновательных когнитивных тренажеров. Игроки заходят в одну комнату, получают общие стимулы и соревнуются в точности и скорости реакции.

Сервер управляет игровым состоянием: генерирует стимулы, проверяет ответы, применяет штрафы, ускоряет игру и определяет победителя. Клиент только отображает состояние и отправляет намерение игрока.

## Ключевые возможности

| Возможность | Описание |
| --- | --- |
| Комнаты на 2-4 игрока | Несколько игроков участвуют в одной синхронизированной игровой сессии. |
| Несколько тренажеров | Поддерживаются N-back, Recent-5, Go / No-Go, Reaction Time и Stroop Test. |
| Серверные стимулы | Стимулы генерируются на сервере и одинаковы для всех игроков. |
| Server-authoritative модель | Сервер проверяет каждый ответ и хранит единственный источник истины. |
| Система ошибок | Ошибки считаются отдельно для каждого игрока. Каждая третья ошибка ускоряет игру для всех. |
| Метрики по игроку | Accuracy, reaction time, false positives, misses, false starts и conflict errors считаются на сервере. |
| Общий темп игры | Все игроки видят один и тот же индекс стимула и один интервал. |
| Подсчёт победителя | Побеждает игрок с наибольшим числом очков; при равенстве учитывается меньшее число ошибок. |
| ELO и ранги | После завершения матча рейтинг игрока обновляется, а изменения сохраняются в историю. |
| Авторизация | Аккаунты и сессии обслуживаются через better-auth. |
| Руководство в приложении | Для каждого тренажера есть отдельная страница с правилами игры. |

## Быстрый старт

### Требования

- Node.js 20+
- npm
- SQLite / libSQL

### Установка

```bash
npm install
```

### Переменные окружения

Создайте `.env.local`:

```bash
DATABASE_URL="file:./data/nback.sqlite"
BETTER_AUTH_SECRET="replace-with-a-long-random-secret"
BETTER_AUTH_URL="http://localhost:3000"
NEXT_PUBLIC_APP_URL="http://localhost:3000"
```

### Миграции

```bash
npm run db:migrate
```

### Запуск в разработке

```bash
npm run dev
```

Откройте:

```text
http://localhost:3000
```

### Production-сборка

```bash
npm run build
npm start
```

## Обзор потока

1. Авторизованный пользователь создаёт комнату и выбирает нужный тренажер.
2. Другие игроки присоединяются к комнате.
3. Владелец комнаты запускает игру.
4. Сервер генерирует и продвигает последовательность стимулов или таймингов.
5. Клиенты получают актуальное состояние и отображают текущий стимул.
6. Игроки отправляют ответы.
7. Сервер проверяет ответы, обновляет очки и метрики, применяет штрафы и рассылает изменения.
8. Игра заканчивается после последнего стимула или ручного завершения владельцем.
9. Сервер рассчитывает и сохраняет результаты.
